Team Standup & Sprint Tracker

Project High-Level Design and code templates

Spring Boot 3.3 Angular 18 PostgreSQL 16 JWT Auth Flyway Caffeine Cache Angular Material Chart.js

Project Overview

This is a full-stack agile project management tool that enables engineering teams to run daily standups, manage sprint backlogs, and track velocity over time. It replaces scattered Slack messages and spreadsheets with a purpose-built board.

High-Level Design (HLD)

System Architecture Layers

Angular 18 SPA (localhost:4200)
↕ HTTP + Bearer Token
Spring Boot REST Controllers (localhost:8080)

Service Layer (Business Logic + Caching)
↓
Spring Data JPA Repositories
↓

Backend Architecture (Spring Boot)

Package Structure

com.sprinttracker/
├── SprintTrackerApplication.java     // @SpringBootApplication entry point
├── controller/
│   ├── AuthController.java           // POST /api/auth/register, /login
│   ├── TeamController.java           // CRUD for teams + members
│   ├── SprintController.java         // CRUD for sprints + velocity
│   ├── TaskController.java           // CRUD for tasks
│   └── StandupController.java        // Post standup, board view
├── service/
│   ├── AuthService.java              // Register/login, BCrypt, JWT generation
│   ├── TeamService.java              // Team CRUD + role-based access control
│   ├── SprintService.java            // Sprint lifecycle + @Cacheable velocity
│   ├── TaskService.java              // Task CRUD + cache eviction trigger
│   ├── StandupService.java           // Upsert standup entry
│   └── BlockerDetectionService.java  // Keyword-based blocker flagging
├── entity/                           // JPA @Entity classes
├── repository/                       // Spring Data JPA interfaces
├── dto/                              // Request/Response records
├── security/                         // JWT filter, config, UserDetails
├── config/                           // CacheNames constants
└── common/                           // GlobalExceptionHandler, custom exceptions

Database Schema Design

Entity Relationship

users (1) ──────┬──── (N) team_members (N) ────┬──── (1) teams
                │                                │
                │                                ├──── (N) sprints
                │                                │            │
                │                                │     (1) ───┼──── (N) tasks
                │                                │            │
                └── (N) standup_entries ──────────┴────────────┘
                        (unique: user + sprint + date)
