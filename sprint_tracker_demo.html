How blocker detection works: BlockerDetectionService.isBlocked() first checks if blockersText is non-empty. If not, it lowercases yesterday + today text and scans for 16 keyword phrases ("blocked", "stuck", "waiting on", "need help", etc.). The "blocked" in "I am blocked on API access" triggers the flag. This is computed on write and stored as a boolean — no re-scanning on reads.

Caching in action: The first request to GET /api/teams/{id}/velocity computes the aggregate from DB (sumPlannedPoints + sumCompletedPoints per sprint). The result is cached in Caffeine with key=teamId. Subsequent requests return instantly from cache until a task is created or updated, which triggers @CacheEvict. The 60s TTL is a safety net.
